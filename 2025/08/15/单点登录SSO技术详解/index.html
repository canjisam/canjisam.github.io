title: 单点登录(SSO)技术详解：原理、实现与最佳实践
date: 2023-11-17 10:00:00

  • 认证授权
    tags:
  • 单点登录
  • 企业应用

简单来说，SSO解决了用户在访问企业内多个应用时需要重复登录的痛点，提高了用户体验和工作效率。

  1. 减少密码疲劳：用户只需记住一组凭据
  2. 简化管理：IT部门可以集中管理用户权限和访问控制

|——|———|————-|
| 凭据存储 | 分散在各个应用中 | 集中存储在认证服务器 |
| 安全控制 | 分散，难以统一管理 | 集中，易于实施安全策略 |

  1. 重定向到SSO服务器：应用A将用户重定向到SSO认证服务器
  2. 生成票据(Ticket)：认证成功后，SSO服务器生成认证票据
  3. 验证票据：应用A验证票据的有效性，确认用户身份
  4. 访问应用B：当用户访问应用B时，重复步骤2-7，但由于用户已在SSO服务器认证，无需再次输入凭据

2.2 SSO架构模型

2.2.1 基于Cookie的SSO

这是最简单的SSO实现方式，适用于同域名下的应用：

  1. 用户登录主应用后，服务器在根域名下设置Cookie

2.2.2 基于令牌的SSO

适用于跨域应用的SSO实现：

  1. 用户登录SSO服务器后获取令牌(Token)

  2. 应用验证令牌有效性，确认用户身份
    优点：支持跨域应用

  3. SP生成SAML请求，重定向到身份提供商(IdP)

  4. IdP生成SAML断言(Assertion)，重定向回SP

缺点：实现复杂，XML处理开销大

2.2.4 基于OAuth/OpenID Connect的SSO

结合OAuth 2.0和OpenID Connect实现的现代SSO方案：

  1. 用户访问客户端应用
  2. 用户在授权服务器认证
  3. 客户端使用授权码获取访问令牌和ID令牌

缺点：配置相对复杂

2.3 SSO流程图示

3.1 CAS (Central Authentication Service)

3.1.1 CAS协议流程

  1. 用户访问应用，应用重定向到CAS服务器
  2. CAS服务器生成票据(TGT)，并发放服务票据(ST)
  3. 应用通过后端通道验证ST的有效性
  • CAS Client：集成到应用中的客户端库
  • ST (Service Ticket)：一次性票据，用于应用验证用户身份

3.1.3 Java实现示例

服务端配置(application.properties):
server.port=8443
server.ssl.key-store-password=changeit

cas.authn.accept.users=casuser::Mellon
客户端集成(Spring Boot):

@EnableCasClient
@SpringBootApplication
public static void main(String[] args) {
}

配置类:
@Configuration
public class SecurityConfig extends WebSecurityConfigurerAdapter {
@Bean
provider.setUserDetails(new SAMLUserDetailsServiceImpl());
return provider;

}

3.3.1 OAuth 2.0角色

  • 客户端(Client)：第三方应用
  • 资源服务器(Resource Server)：托管受保护资源

3.3.2 OpenID Connect扩展

OpenID Connect在OAuth 2.0基础上增加了：

  • UserInfo端点：获取用户详细信息

3.4 JWT (JSON Web Token)在SSO中的应用

JWT常用于实现基于令牌的SSO，特别是在微服务架构中。

3.4.1 JWT结构

  • Payload：包含声明(Claims)，如用户ID、角色等
  • Signature：确保令牌完整性的签名
  1. 用户登录认证服务器
  2. 客户端存储JWT(通常在localStorage或Cookie中)
  3. 应用验证JWT签名和有效期，提取用户信息

3.4.3 Spring Boot实现JWT认证

4.1 企业级SSO解决方案

4.1.1 Keycloak

Keycloak是一个开源的身份和访问管理解决方案，支持多种标准协议。
特点:

  • 多租户支持

## 7. SSO的未来趋势
### 7.1 无密码认证
- 生物识别技术(指纹、面部识别)
- 魔术链接(Magic Links)和一次性密码
### 7.2 自适应认证
- 基于风险的认证策略
- 上下文感知认证
### 7.3 去中心化身份(DID)
- 自主身份(Self-sovereign Identity)
- 可验证凭证(Verifiable Credentials)
## 8. 总结
单点登录(SSO)技术通过允许用户使用单一凭据访问多个应用，极大地提升了用户体验和安全性。本文详细介绍了SSO的工作原理、主要实现技术(CAS、SAML、OAuth/OIDC)、企业级解决方案以及最佳实践。
随着身份管理的重要性不断提升，SSO技术将继续演进，融合更多创新元素，如无密码认证、自适应认证和去中心化身份。企业在实施SSO时，应根据自身需求选择合适的解决方案，并注重安全性、用户体验和可扩展性。
